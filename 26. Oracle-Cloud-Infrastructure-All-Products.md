## 1. Compute

### 1.1 Compute Instances (Virtual Machines)
- **Description:** Scalable virtual machines running on OCI's high-performance bare-metal hardware.
- **Shapes:** Standard (E-series AMD, X9 Intel), Optimized, GPU, HPC, Flex shapes (configurable OCPU/memory).
- **Key Features:** Live migration, in-transit/at-rest encryption, burstable instances (E3/E4.Flex), cloud-init customization.
- **Use Cases:** Web servers, application servers, custom workloads requiring OS-level control.

### 1.2 Bare Metal Instances
- **Description:** Dedicated physical servers with no hypervisor overhead.
- **Key Features:** Full hardware isolation, high IOPS NVMe SSD, 100 Gbps network, ideal for latency-sensitive workloads.
- **Use Cases:** High-frequency trading, data warehousing, scientific computing, VMware migration.

### 1.3 GPU Instances
- **Description:** Instances with NVIDIA GPUs (A10, A100, H100 series).
- **Key Features:** NVLink, high-bandwidth GPU memory, multi-GPU configurations.
- **Use Cases:** AI/ML training and inference, visual rendering, simulation.

### 1.4 HPC Instances
- **Description:** High-Performance Computing nodes interconnected with RDMA clustering network.
- **Key Features:** Ultra-low latency cluster networking (RDMA over Converged Ethernet), co-located placement groups.
- **Use Cases:** Computational fluid dynamics, molecular dynamics, weather modeling.

### 1.5 Autoscaling
- **Description:** Automatically adjusts the number of compute instances based on defined metrics or schedules.
- **Types:** Metric-based (CPU/memory thresholds), Schedule-based.
- **Use Cases:** Handling traffic spikes without manual intervention.

### 1.6 Instance Pools
- **Description:** Manages a collection of compute instances as a group using the same configuration.
- **Key Features:** Integrated with load balancers and autoscaling, consistent instance lifecycle management.

### 1.7 Dedicated Virtual Machine Hosts
- **Description:** Single-tenant physical servers where you run your VMs exclusively.
- **Use Cases:** Regulatory compliance, bring-your-own-license (BYOL) workloads.

### 1.8 Burstable Instances
- **Description:** Instances billed on a baseline OCPU with the ability to burst to full capacity.
- **Use Cases:** Dev/test environments, low-traffic workloads with occasional spikes.

### 1.9 Preemptible Instances
- **Description:** Short-lived, low-cost instances reclaimed by OCI when capacity is needed.
- **Use Cases:** Batch jobs, fault-tolerant workloads, CI/CD pipelines.

### 1.10 OS Management Service
- **Description:** Automates OS patching, updates, and package management across instances.
- **Supports:** Oracle Linux, Windows Server.

## 2. Storage

### 2.1 Block Volume
- **Description:** Persistent, durable, high-performance block storage attached to compute instances.
- **Performance Tiers:** Lower Cost, Balanced, Higher Performance, Ultra-High Performance (UHP).
- **Key Features:** Online resize, volume groups, cross-region replication, encryption at rest, backup policies.
- **Use Cases:** Boot volumes, database storage, enterprise applications.

### 2.2 Object Storage
- **Description:** Scalable, durable object storage for unstructured data.
- **Tiers:** Standard (hot), Infrequent Access, Archive.
- **Key Features:** S3-compatible API, lifecycle policies, versioning, pre-authenticated requests (PARs), replication, event triggers.
- **Use Cases:** Backup/restore, data lakes, media storage, log archiving.

### 2.3 Archive Storage
- **Description:** Lowest-cost storage for rarely accessed data with retrieval times of hours.
- **Key Features:** 90-day minimum retention, restore time ~1–4 hours.
- **Use Cases:** Long-term compliance archives, disaster recovery cold copies.

### 2.4 File Storage (NFS)
- **Description:** Managed NFS v3 file system accessible from multiple compute instances simultaneously.
- **Key Features:** Auto-scaling, snapshot support, AD-scoped mount targets, LDAP/Kerberos integration.
- **Use Cases:** Shared file systems, content management, HPC scratch directories.

### 2.5 Local NVMe SSD Storage
- **Description:** Ephemeral, ultra-fast NVMe SSDs physically attached to specific compute shapes (DenseIO, HPC).
- **Use Cases:** Temporary high-speed scratch space, caching tiers.

### 2.6 Data Transfer Service
- **Description:** Secure physical data transfer to/from OCI using shipping hard drives or appliances.
- **Use Cases:** Initial large-scale data migration to object storage when network transfer is impractical.

### 2.7 Storage Gateway
- **Description:** On-premises cache gateway that connects to OCI Object Storage via NFS or SMB.
- **Use Cases:** Hybrid cloud storage extension, backup from on-premises to OCI.

## 3. Networking

### 3.1 Virtual Cloud Network (VCN)
- **Description:** Software-defined private network within OCI. Logically isolated, customer-controlled.
- **Components:** Subnets (public/private), route tables, security lists, DHCP options.
- **Use Cases:** Foundation for all OCI deployments.

### 3.2 Subnets
- **Types:** Regional (span all ADs in a region) or AD-specific.
- **Security:** Security Lists (stateful firewall rules), Network Security Groups (NSGs).

### 3.3 Internet Gateway
- **Description:** Allows public internet access to/from resources in a public subnet.

### 3.4 NAT Gateway
- **Description:** Enables outbound internet access for private subnet resources without exposing them.

### 3.5 Service Gateway
- **Description:** Provides private access to OCI public services (Object Storage, ADW, etc.) without traversing the internet.

### 3.6 Local Peering Gateway (LPG)
- **Description:** Connects two VCNs within the same region.

### 3.7 Dynamic Routing Gateway (DRG)
- **Description:** Virtual router enabling traffic between VCNs and on-premises networks (FastConnect, VPN).
- **Supports:** Transit routing, hub-and-spoke network topologies.

### 3.8 VPN Connect (Site-to-Site VPN)
- **Description:** IPSec VPN tunnels between on-premises networks and OCI.
- **Key Features:** Redundant tunnels, BGP or static routing.

### 3.9 FastConnect
- **Description:** Dedicated, private, high-bandwidth connection between on-premises data centers and OCI (equivalent to AWS Direct Connect / Azure ExpressRoute).
- **Bandwidth:** 1 Gbps to 10 Gbps, partner-based or colocation.
- **Use Cases:** Low-latency hybrid workloads, large data transfers.

### 3.10 Load Balancer
- **Description:** Fully managed Layer 7 (HTTP/HTTPS) load balancer.
- **Features:** SSL termination, URL-based routing, health checks, WAF integration, session persistence.
- **Shapes:** Flexible (10 Mbps to 8 Gbps).
- **Use Cases:** Web application traffic distribution.

### 3.11 Network Load Balancer (NLB)
- **Description:** Layer 4 (TCP/UDP) ultra-low-latency load balancer.
- **Key Features:** Preserves source IP, supports UDP, no SSL termination overhead.
- **Use Cases:** High-throughput applications, network virtual appliances.

### 3.12 DNS
- **Description:** Managed DNS service (public and private zones).
- **Features:** Geo-steering, ASN steering, health-check-based traffic management.

### 3.13 Traffic Management Steering Policies
- **Description:** Global traffic management using routing policies (round-robin, failover, geolocation).

### 3.14 Web Application Firewall (WAF)
- **Description:** Edge-based protection against Layer 7 attacks.
- **Features:** OWASP rule sets, rate limiting, bot management, custom rules.
- **Use Cases:** DDoS mitigation, API protection.

### 3.15 Bastion Service
- **Description:** Fully managed bastion host providing secure SSH/RDP access to private resources.
- **Key Features:** Session auditing, time-limited sessions, no public IP on target instance required.

### 3.16 Network Firewall
- **Description:** Managed next-generation firewall powered by Palo Alto Networks technology.
- **Features:** IDPS, URL/app filtering, SSL inspection, threat intelligence.

### 3.17 Secure Desktops (DaaS)
- **Description:** Virtual desktop infrastructure service.
- **Use Cases:** Remote workforce, BYOD security.

## 4. Database

### 4.1 Oracle Base Database Service (VM / BM)
- **Description:** Managed Oracle Database on VMs or Bare Metal with full DBA access.
- **Editions:** Standard Edition 2, Enterprise Edition, EE Extreme Performance.
- **Features:** Automatic backups, Data Guard standby, patch automation, BYOL.
- **Use Cases:** OLTP, mixed workloads, lift-and-shift from on-premises.

### 4.2 Oracle Exadata Database Service (Dedicated)
- **Description:** Fully managed Exadata on OCI with dedicated infrastructure.
- **Features:** Smart Scan, Storage Indexes, Database In-Memory, HCC compression.
- **Use Cases:** Mission-critical, high-performance OLTP, data warehousing.

### 4.3 Oracle Exadata Database Service (Cloud@Customer)
- **Description:** Exadata hardware in the customer's data center managed by Oracle.
- **Use Cases:** Data sovereignty, ultra-low latency to on-premises apps.

### 4.4 Autonomous Database — Shared (ADB-S)
- **Sub-products:**
  - **Autonomous Transaction Processing (ATP):** Optimized for OLTP.
  - **Autonomous Data Warehouse (ADW):** Optimized for analytics.
  - **Autonomous JSON Database (AJD):** Document-centric workloads.
  - **APEX Application Development (APEX Service):** Low-code app platform.
- **Features:** Self-tuning, self-patching, self-securing, auto-scaling compute/storage, in-database ML.
- **Connectivity:** mTLS wallet-based or TLS direct (no wallet required since 2021).

### 4.5 Autonomous Database — Dedicated (ADB-D)
- **Description:** Autonomous Database on dedicated Exadata Cloud Infrastructure.
- **Use Cases:** Regulatory compliance, predictable performance, full isolation.

### 4.6 MySQL HeatWave
- **Description:** Managed MySQL service with HeatWave in-memory query accelerator for real-time analytics.
- **Features:** HeatWave Lakehouse (query object storage), HeatWave AutoML, HeatWave GenAI.
- **Use Cases:** OLTP + analytics in a single MySQL system, eliminates ETL to a separate warehouse.

### 4.7 NoSQL Database Cloud Service
- **Description:** Fully managed, serverless NoSQL (key-value, document, table) with predictable single-digit millisecond latency.
- **Use Cases:** IoT, user profiles, real-time personalization, session management.

### 4.8 GoldenGate (Cloud Service)
- **Description:** Managed real-time data replication and stream data processing service.
- **Features:** Low-latency CDC (change data capture), multi-source/target, heterogeneous replication.
- **Use Cases:** Zero-downtime migration, real-time analytics feeds, disaster recovery replication.

### 4.9 Database Migration Service
- **Description:** Automated tool for migrating Oracle and non-Oracle databases to OCI.
- **Supports:** Oracle → Autonomous DB, MySQL → MySQL HeatWave, PostgreSQL.

### 4.10 Database Management Service
- **Description:** Unified console for monitoring, performance tuning, and fleet management of Oracle databases.
- **Features:** Performance Hub, AWR analysis, tablespace management.

### 4.11 Operations Insights
- **Description:** ML-based resource planning and performance analysis for databases and hosts.
- **Features:** SQL Explorer, capacity planning forecasting, AWR hub.

## 5. Security

### 5.1 Cloud Guard
- **Description:** Cloud security posture management (CSPM) service that detects misconfigurations and threats.
- **Features:** Detectors (config/activity rules), Responders (auto-remediation), Security Score.
- **Use Cases:** Continuous compliance monitoring, threat detection.

### 5.2 Security Zones
- **Description:** Enforce security policies (deny public buckets, deny unencrypted volumes) at the compartment level.
- **Use Cases:** Preventive controls for sensitive compartments.

### 5.3 Vault (Key Management)
- **Description:** Managed HSM-backed key management service.
- **Types:** Virtual Private Vault (dedicated HSM), Default Vault (shared).
- **Key Types:** AES (symmetric), RSA/ECDSA (asymmetric).
- **Features:** Customer-managed encryption keys (CMEK), key rotation, secrets management.
- **Use Cases:** Encrypting volumes, object storage, DB, bring your own key (BYOK).

### 5.4 Certificates Service
- **Description:** Managed TLS/SSL certificate lifecycle management.
- **Features:** Public CA-signed and private CA certificates, auto-renewal, integration with Load Balancer/API GW.

### 5.5 Web Application Firewall (WAF)
- See [Networking → WAF](#314-web-application-firewall-waf).

### 5.6 DDoS Protection
- **Description:** Always-on network-level DDoS protection included with OCI at no extra cost.

### 5.7 Threat Intelligence Service
- **Description:** Aggregates threat indicators from Oracle and community feeds to enrich security events.

### 5.8 Data Safe
- **Description:** Unified security control center for Oracle Databases.
- **Features:** Security assessment, user assessment, data discovery (sensitive data), data masking, activity auditing.
- **Use Cases:** Compliance reporting (GDPR, HIPAA), database security hardening.

### 5.9 Vulnerability Scanning
- **Description:** Scans compute instances and container images for OS-level and application vulnerabilities.
- **Features:** CVE detection, CIS benchmarks, integration with Cloud Guard.

### 5.10 Security Advisor
- **Description:** Workflow-based tool guiding users to configure secure architectures (combines WAF, Vault, VCN, Identity).

### 5.11 Bastion Service
- See [Networking → Bastion](#315-bastion-service).

## 6. Identity and Access Management

### 6.1 IAM (Identity and Access Management)
- **Description:** Core OCI service for authentication and authorization.
- **Concepts:**
  - **Users:** Human or service accounts.
  - **Groups:** Collections of users.
  - **Policies:** Human-readable statements granting permissions.
  - **Compartments:** Logical containers for organizing resources.
  - **Dynamic Groups:** Groups of resources (instances, functions) for instance-principal auth.
  - **Tags:** Freeform and defined tags for governance/cost tracking.

### 6.2 Identity Domains
- **Description:** Next-generation IAM built on Oracle Identity Cloud Service (IDCS), now integrated into OCI.
- **Features:** SCIM provisioning, SAML 2.0 / OIDC federation, MFA, adaptive security, self-service.
- **Types:** Free, Oracle Apps, Oracle Apps Premium, Premium, External User.

### 6.3 Federation
- **Description:** Integrate with external identity providers (Microsoft AD, Okta, Google) via SAML 2.0.
- **Use Cases:** SSO into OCI console.

### 6.4 Instance Principals
- **Description:** Allow OCI compute instances to authenticate with IAM without hard-coded credentials.

### 6.5 Resource Principals
- **Description:** Allow OCI functions, data-flow apps, and other resources to authenticate with IAM.

### 6.6 Compartments
- **Description:** Logical grouping of OCI resources. Support nesting (up to 6 levels) and policy inheritance.
- **Use Cases:** Environment isolation (dev/test/prod), cost allocation, team separation.

## 7. Containers and Kubernetes

### 7.1 Oracle Kubernetes Engine (OKE)
- **Description:** Managed Kubernetes service.
- **Node Types:** Managed Nodes (OCI-managed worker nodes), Virtual Nodes (serverless pods — no node management), Self-Managed Nodes.
- **Key Features:** Cluster autoscaler, node pool management, OCI CNI plugin (VCN-native pods), Workload Identity, ARM support.
- **Use Cases:** Container orchestration, microservices, CI/CD workloads.

### 7.2 Container Instances
- **Description:** Serverless containers without Kubernetes overhead — OCI manages the infrastructure.
- **Use Cases:** Short-lived batch jobs, simple containerized APIs, testing, CI tasks.

### 7.3 Container Registry (OCIR)
- **Description:** Managed Docker-compatible container image registry.
- **Features:** Private/public repos, image signing (Notary), vulnerability scanning, tag-based retention policies.
- **Use Cases:** Store and manage Docker images for OKE, Container Instances, Functions.

### 7.4 Artifact Registry
- **Description:** Managed repository for software artifacts: container images, Helm charts, generic artifacts (JARs, ZIPs, npm packages).
- **Key Features:** OCIR consolidation (Helm charts), version-controlled artifacts.
- **Use Cases:** DevOps artifact management.

## 8. Developer Services

### 8.1 OCI DevOps
- **Description:** End-to-end CI/CD platform native to OCI.
- **Components:**
  - **Code Repository:** Managed Git (private).
  - **Build Pipelines:** Managed build runners, multi-stage builds, build specs (YAML).
  - **Artifact Repository:** Store build outputs.
  - **Deployment Pipelines:** Deploy to OKE, Compute, Functions, Container Instances.
  - **Environments:** Define target deployment environments.
- **Use Cases:** Full CI/CD lifecycle for OCI-native applications.

### 8.2 Resource Manager (Terraform-as-a-Service)
- **Description:** Managed Terraform service for infrastructure-as-code on OCI.
- **Features:** Stacks, plans, apply/destroy, drift detection, private endpoint support, Git integration.
- **Use Cases:** Consistent, repeatable infrastructure provisioning.

### 8.3 Notifications Service
- **Description:** Pub/sub broadcast messaging for sending alerts to humans or services.
- **Protocols:** Email, Slack (via HTTPS), PagerDuty, OCI Functions, OCI Queue.
- **Use Cases:** Alerting pipelines, operational notifications.

### 8.4 Events Service
- **Description:** Captures state changes on OCI resources and routes them to actions.
- **Sources:** Object Storage, Compute, Database events, etc.
- **Targets:** Functions, Notifications, Streaming, Queue.
- **Use Cases:** Event-driven automation (e.g., trigger function on S3 PUT).

### 8.5 Scheduled Jobs (Resource Scheduler)
- **Description:** Schedule start/stop of OCI resources (instances, DB systems) on a time-based schedule.
- **Use Cases:** Cost savings in dev/test environments.

### 8.6 Service Connector Hub
- **Description:** Data movement pipeline routing data between OCI services.
- **Sources:** Logging, Monitoring, Streaming, Object Storage, Connector Hub queue.
- **Targets:** Object Storage, Streaming, Functions, Logging Analytics, Notifications, Monitoring.
- **Use Cases:** Centralized log pipeline, metric routing, event fan-out.

### 8.7 API Gateway
- See [Section 10](#10-api-management).

### 8.8 Cloud Shell
- **Description:** Browser-based shell with pre-installed OCI CLI, kubectl, Terraform, SDKs.
- **Use Cases:** Quick OCI management tasks without local tooling setup.

### 8.9 OCI SDK and CLI
- **Languages:** Java, Python, TypeScript/JavaScript, Go, .NET, Ruby.
- **Key Features:** Pagination, waiters, retry logic, instance/resource principal auth.

## 9. Functions (Serverless)

### 9.1 OCI Functions
- **Description:** Serverless, event-driven function-as-a-service based on the open-source **Fn Project**.
- **Runtimes:** Java, Python, Node.js, Go, Ruby, Docker (custom container image).
- **Invocation:** HTTP (API Gateway), Events, Notifications, Service Connector Hub.

- **Use Cases:** Lightweight microservices, event handlers, automation tasks, data transformation.

### 9.2 Application (Functions Container)
- **Description:** Logical container grouping related functions; shares config, VCN subnet, and logging.

## 11. Messaging and Streaming

### 11.1 Streaming Service
- **Description:** Managed, Apache Kafka-compatible publish/subscribe data streaming service.
- **Key Features:** Partitioned streams, consumer groups, 7-day default retention (up to 168 hours), Kafka API compatibility.
- **Use Cases:** Real-time event pipelines, IoT telemetry, log streaming, decoupling microservices.

### 11.2 Queue Service
- **Description:** Fully managed message queue service (point-to-point) for decoupling producers and consumers.
- **Key Features:** At-least-once delivery, visibility timeout, dead-letter queue (DLQ), FIFO ordering within channels.
- **Use Cases:** Task queues, order processing, work item distribution.

### 11.3 Notifications Service
- See [Developer Services → Notifications](#83-notifications-service).

## 12. Integration Services

### 12.1 Oracle Integration Cloud (OIC)

  - **File Server:** Managed sFTP server for file-based integrations.
- **Use Cases:** ERP/CRM integration, order-to-cash automation, B2B EDI.

### 12.2 API Platform Cloud Service
- **Description:** Enterprise API lifecycle management (design, publish, govern, analyze) — often superseded by OCI API Gateway + Portal.

### 12.3 GoldenGate (Streaming Integration)

### 12.4 Data Integration Service

- **Description:** Metadata labels (freeform or defined namespaces with validation) applied to resources.

## Key OCI Concepts at a Glance

| Concept | Description |
|---|---|
| **Region** | Geographic area containing one or more Availability Domains (ADs) |
| **Availability Domain (AD)** | Isolated data center within a Region |
| **Fault Domain (FD)** | Logical grouping within an AD to protect against hardware failure |
| **Compartment** | Logical container for resources; used for access control and billing |
| **OCID** | Oracle Cloud Identifier — unique ID for every OCI resource |
| **Tenancy** | Root compartment; top-level account |
| **Shape** | VM/BM specification defining OCPU, memory, network bandwidth |
| **OCPU** | Oracle CPU = 1 physical core (2 vCPUs equivalent) |
| **Free Tier** | Always-free resources (2 AMD VMs, 2 Autonomous DBs, Object Storage, etc.) |
| **OCI CLI** | Command-line interface for managing OCI resources |
| **Terraform Provider** | Official OCI provider for infrastructure-as-code |
